# 即梦（Jimeng/Dreamina）· 生图提示词技法卡

> 源自一支开幕式宣传片上千张图像生成的实战沉淀。即梦吃**中文原生**，提示词要精简、画面感强、别堆砌。

## 1. 生图公式

```
主体 + 场景 + 动作 + 风格 + 氛围/光线
```

一句话式书写，每个槽位用具体可见的描述填满：

```
[主体：谁/什么，外观特征逐项写]在[场景：哪里，环境要素]中[动作：可见的姿态动作]，[风格：渲染风格+质感等级]，[氛围/光线：光源方向+色调，用具体色名]。
```

**示例**（占位角色）：

```
一位戴黑框眼镜的年轻馆员与一只圆头大眼的蜜蜂吉祥物并肩站在儿童剧场舞台中央向观众挥手，馆员在左吉祥物在右，高端3D卡通动画电影质感，皮克斯式角色塑形，暖金舞台顶光加柔和轮廓光，橙金主调节日氛围，16:9。
```

## 2. 五条即梦实战规则

1. **中文原生直写**：不必翻译成英文；中文的画面词（"暖金黄昏光""发光立体书花"）即梦理解得很好。
2. **抽象词换具象**："梦幻""有节奏感"这类词不稳定，改写为可见之物（"漫天细腻金光点""书页微微掀起"）。
3. **角色特征逐项重复**：模型没有记忆，每张图都要把角色外观从头到脚写全（发型/服饰/配色/标志物/比例）；多角色加"身份独立，特征绝不互串"。
4. **画面内不写字**：需要文字的画面（海报、标题板）让画面留白，正式文字后期添加——模型画字必崩。
5. **省字数**：平台把空格、换行也计入字数且计数偏严；删装饰符号，省下的字写更细的画面描述。

## 3. 多版本比选工作法

- 同一分镜生成 2–3 版：可变化**光线时段**（黄昏金/日间白/夜晚蓝）或**核心意象**（同一主题的不同视觉方案），其余槽位不动；
- 按与整体基调、目标受众审美、叙事连贯性的契合度**人工定版**——筛选与匹配是人的业务判断，不要外包给模型；
- 已定版图像一律复用，不重复生成。

## 4. 图像先行（衔接视频生成）

即梦生图的最终用途常是**图生视频的首帧**：定版图像以确定的构图、造型与色彩为视频生成锁定视觉边界。规则：图不定版，不进视频——不确定性在图像层每多消化一分，视频层的返工便少一分。
